# First Week Workflow

Use this workflow when a user is entering a new job, internship, team, or project.

## Day 0

Run:

```text
/onboard
```

Input:

- role description
- manager expectations
- onboarding docs
- calendar for the first week
- people already known

Output:

- role map
- first-week plan
- people map
- question backlog
- manager alignment message

## Days 1-3

After each important conversation, run:

```text
/meeting
```

Then update:

- people map
- decision log
- task board
- open questions

## Day 4

Run:

```text
/stakeholder
```

Use it to identify high-influence people, user groups, collaborators, and people who need updates.

## Day 5

Run:

```text
/update
/retro
```

Produce a manager-ready weekly update and choose next week's top three priorities.
