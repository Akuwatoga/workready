# Status Report Method

Use this method when the user needs to report project progress, risk, blockers, or decisions.

## Purpose

Make the true state of work visible so managers and collaborators can decide, support, or unblock.

## Status Levels

| Level | Meaning | When to use |
|---|---|---|
| Green | On track | Scope, timeline, and quality are stable |
| Yellow | At risk | One key factor may affect delivery |
| Red | Blocked or off track | A decision, resource, or major issue prevents delivery |

## Report Sections

1. Overall status
2. One-line summary
3. Completed work and evidence
4. Current work
5. Next actions
6. Risks and blockers
7. Decisions needed
8. Support requested

## Operating Rules

- Report outcomes and evidence, not only effort.
- Make the ask explicit.
- Put the most important risk near the top.
- Name owners and dates.
- Include a recommendation when asking for a decision.
- Keep the report short enough for a manager to scan in under one minute.

## Escalation Pattern

```text
Status: Yellow/Red
Issue:
Impact:
Options:
Recommendation:
Decision needed by:
Owner:
```
