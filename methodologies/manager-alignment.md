# Manager Alignment Method

Use this method whenever the user needs to understand a manager's expectations, priorities, communication style, or decision rules.

## Purpose

Turn an unclear manager relationship into a predictable operating relationship.

## Alignment Areas

| Area | Questions to clarify |
|---|---|
| Success criteria | What would make the next week, month, or quarter successful? |
| Priorities | Which work matters most if everything cannot be done? |
| Quality bar | What does "good enough" look like for this deliverable? |
| Cadence | How often should progress be reported? |
| Format | Does the manager prefer chat, email, docs, meetings, or dashboards? |
| Decisions | Which decisions can the user make, and which need approval? |
| Escalation | When should risks or blockers be surfaced? |
| Feedback | How should the user request and receive feedback? |

## Operating Rules

- Clarify the outcome, not just the task.
- Ask about priority tradeoffs before overcommitting.
- Confirm assumptions in writing.
- Make risks visible early.
- Bring recommendations, not only problems.
- Keep manager updates short, structured, and action-oriented.

## Manager Message Pattern

```text
I want to confirm my understanding:

Goal:
What I will do:
What I will not cover yet:
Output:
Timeline:
Risks or dependencies:
Question/decision needed:
```

## Red Flags

- The user has more than three top priorities.
- There is no named decision maker.
- The deadline is fixed but scope is unclear.
- The user is doing work without knowing how it will be judged.
- The manager only hears about blockers after the deadline is at risk.
