# Business/Data Map Method

Use this method when the user needs to understand a workflow, report, dashboard, dataset, spreadsheet, CRM process, or automation opportunity.

## Purpose

Convert messy business operations into a clear map of processes, data sources, owners, pain points, and AI automation opportunities.

## Layers

| Layer | Questions |
|---|---|
| Business process | What happens, in what order, and who does it? |
| Data source | Where does the information come from? |
| Data quality | What is missing, duplicated, stale, manual, or inconsistent? |
| Reporting | Which outputs depend on this data? |
| Decisions | Who uses the output to make decisions? |
| Automation | Which steps can be assisted, checked, summarized, routed, or generated? |

## Operating Rules

- Start with the business workflow before designing automation.
- Identify source of truth separately from copied spreadsheets.
- Name data owners.
- Mark access constraints and permission gaps.
- Prioritize automation by value, frequency, risk reduction, and implementation difficulty.
- Separate "AI can draft" from "AI can decide"; keep human approval for high-risk decisions.

## AI Opportunity Types

| Type | Example |
|---|---|
| Summarize | Turn notes or records into a status update |
| Extract | Pull fields from emails, PDFs, forms, or chats |
| Reconcile | Compare two sources and flag mismatches |
| Classify | Categorize tickets, leads, issues, or requests |
| Draft | Generate replies, reports, requirements, or SOPs |
| Monitor | Watch for stale data, missing owners, or deadline risk |
| Route | Send items to the right owner based on rules |
