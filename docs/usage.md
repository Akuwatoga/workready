# Usage

## Basic Pattern

Use a command and paste your raw notes:

```text
/update

Here are my notes:
- finished CRM cleanup
- found 137 duplicate records
- sales ops needs to confirm the merge rule
- dashboard refresh may be delayed
```

The skill should turn the notes into a manager-ready artifact with progress, risks, asks, and next actions.

## First Month Workflow

| Time | Recommended skills | Purpose |
|---|---|---|
| Before day 1 | `/onboard`, `/stakeholder` | Understand role, people, and questions |
| Week 1 | `/onboard`, `/tasks`, `/update` | Build a first-week operating rhythm |
| Week 2 | `/plan`, `/meeting`, `/message` | Convert assignments and meetings into action |
| Week 3 | `/raci`, `/status`, `/datamap` | Clarify ownership, reporting, and systems |
| Week 4 | `/first90`, `/retro` | Turn learning into a 30/60/90 plan |

## Daily Workflow

1. Start the day with `/tasks`.
2. Use `/plan` for vague assignments.
3. Use `/meeting` after every important discussion.
4. Use `/message` before sending sensitive messages.
5. Use `/update` before manager syncs.

## Weekly Workflow

1. Run `/retro` on Friday or Sunday.
2. Convert unfinished work into `/tasks`.
3. Turn project risk into `/status`.
4. Update stakeholder communication plans with `/stakeholder`.
